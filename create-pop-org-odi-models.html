Oracle Data Integrator (ODI) 12c

  • Create ODI Model, Populate the Model and Organize the Model
A model is a set of datastores that go along with the data structures in the physical schema (think of when you set up the topology layer)

To create a model:

  1. In ODI Studio go to the Designer Navigator and expand the Models panel (left side)
  2. Right-click and select New Model
  3. Input the Name, Technology and Logical Schema (on which your model will be based)
  4. Next choose the Reverse Engineering tab and set the Context that will be used and saved

Load the Model with Data (standard way):

  1. Go to the Reverse Engineer tab. Select Standard then select the context to be used
  2. Select the types of objects to reverse engineer
  3. Click on the selective reverse-engineering tab and choose selective reverse engineering, new datastores, existing datastores and objects to reverse engineer
  4. A list of datastores to be reversed engineered appears and choose save
  5. Click reverse engineer from the Model toolbar
The populated datastore will appear under the models panel.

Organizing the Model:

Similar to how we organize our desktops, laptos, etc we can use folders and subfolders to help organize your models. Also, you can use markers too.

Create a model folder
  1. Expand the Models panel
  2. Click New Model Folder in the toolar of the Model panel
  3. Enter a name for your folder and save


You can create sub-models by right-clicking on the model and choosing New sub-model.

There are 2 ways to arrange folders and sub-folders. 1) drag and drop the datastore (model) you want to put into the sub-folder/folder. 2) Set up automatic distribution - this compares datastore names to an automatic assignment mask, if they match this pattern they get moved into this sub-folder.

Set up the automatic distribution of datastores in a sub-model
  1. In the sub-folders (models) distribution tab select Datastore distribution rule and choose the rule
  2. In automatic assignment mask field enter the pattern the datastore names must match to be associated into this sub-model
  3. In the order of mask application after a reverse engineer field enter the order in which rules should be applied at the end of a reverse and click Distribution to apply.
