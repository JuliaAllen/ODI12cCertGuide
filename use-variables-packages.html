Oracle Data Integrator (ODI) 12c

  • Use variables in packages (refresh, startup parameters)


A variable is an object that stores a single value. It can be a data, string or number. It can be used in any expression and in the metadata of the repository.

It is resolved when the command containing it is executed by the agent or GUI

When you create a package, you have to tell it when to do. These are called steps. ODI has different types of steps:
  • Mapping steps
  • Procedure steps
  • Variable steps
  • ODItool steps (mostly OdiStartScen, OdiInvokeWebService, OS Command).


You always want to make sure you declare a variable in ODI. Here is a list of variables to use in a package:

  • Declare Variable: explicitly declares a variable in the package
  • Refresh Variable: refreshes the variable by running the query specified in the variable definition
  • Set Variable:
  • Assign - sets the current value of variable. You can:
    1. Get the value from a SQL Select statement
    2. Manually/explicitly assign the value in a package
    3. Increment the value (only for those with numeric data types)
    4. Assign value at runtime
  • Increment - increases or decreases a numeric value by the specified amount
  • Evaluate Variable: compares the value of the variable with given value according to an operator. Allows for branching.
